  • Для реализации WEB приложения, которое будет предоставлять возможность организациям принимать оплаты с помощью QR кода, используя систему быстрых платежей, можно использовать следующую архитектуру и технологии:
  • Frontend:
    • Язык программирования: JavaScript
    • Фреймворк: ReactJS
    • Библиотека для генерации QR-кода: QRCode.js
    • Для обеспечения безопасности данных можно использовать библиотеки, такие как bcryptjs или CryptoJS.
  • Backend:
    • Язык программирования: любой язык, поддерживающий работу с REST API
    • Фреймворк: Node.js (для примера)
    • СУБД: PostgreSQL (для примера)
    • Библиотека для работы с базой данных: Sequelize
    • Библиотека для генерации и чтения QR-кодов: qr-image
    • Для обеспечения безопасности можно использовать следующие библиотеки: helmet, express-rate-limit, express-validator.
  • API:
    • Стандарт для работы с API: RESTful API
    • Для авторизации и аутентификации пользователей можно использовать токены JWT.
    • Для просмотра истории операций и управления статусами операций можно использовать базу данных и соответствующие запросы.
  • Архитектура:
    • Фронтенд и бэкенд разделены на две части, связанные между собой через API.
    • Для обеспечения безопасности веб-приложения рекомендуется использовать SSL/TLS протоколы для защиты данных, передаваемых между клиентом и сервером.
  • Описание логики работы сервиса:
    • Организация регистрируется в качестве участника СБП через форму регистрации, которая отправляет запрос на бэкенд, где происходит валидация данных и сохранение в базе данных.
    • Пользователь может авторизоваться в системе с помощью логина и пароля или токена JWT, после чего ему будет доступен функционал генерации QR-кода для оплаты.
    • При генерации QR-кода, в него включается необходимая информация об операции, включая сумму, идентификатор организации и другую информацию.
    • Сгенерированный QR-код предоставляется пользователю на экране или отправляется на email или мобильный телефон.
    • При оплате, система проверяет корректность QR-кода и проводит транзакцию. Статус операции меняется на "выполнено
