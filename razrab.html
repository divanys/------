  • Для реализации WEB приложения можно использовать следующую архитектуру и технологии:
  • Frontend:
    • Язык программирования: Для разработки фронтенда можно использовать любой язык программирования, который может работать с сетевыми протоколами (HTTP, WebSocket) и имеет поддержку библиотек для работы с DOM. Как правило, для разработки фронтенда часто используют JavaScript или TypeScript.
    • Фреймворк: В качестве фреймворка для разработки фронтенда можно выбрать любой современный фреймворк, такой как React, Angular, Vue.js и др. Выбор фреймворка зависит от предпочтений команды разработчиков и технических требований проекта.
    • Библиотека для генерации QR-кода: Для генерации QR-кода можно использовать библиотеку QRCode.js или любую другую аналогичную библиотеку, которая поддерживает генерацию QR-кода.
    • Для обеспечения безопасности данных: Для обеспечения безопасности данных можно использовать криптографические библиотеки, такие как bcryptjs или CryptoJS. Эти библиотеки позволяют хранить пароли и другие конфиденциальные данные в зашифрованном виде и обеспечивают защиту от атак, связанных с взломом и утечкой данных.
  • Backend:
    • Язык программирования: Для реализации бэкенда, необходимо выбрать язык программирования, который поддерживает работу с REST API. Хорошим выбором будет использование языков, таких как JavaScript (с помощью Node.js), Python, Java, Ruby или PHP.
    • СУБД: Для работы с базой данных можно использовать любую реляционную СУБД, такую как PostgreSQL, MySQL или SQL Server. В зависимости от выбранной СУБД, можно выбрать соответствующую библиотеку для работы с ней. Например, для работы с PostgreSQL рекомендуется использовать библиотеку Sequelize.
    • REST API: Для создания REST API рекомендуется использовать фреймворки, такие как Express.js, Flask, Spring или Ruby on Rails. Они предоставляют удобный интерфейс для создания и настройки API, а также поддерживают работу с HTTP запросами и ответами.
    • Библиотека для генерации и чтения QR-кодов: Для генерации и чтения QR-кодов можно использовать библиотеки, такие как qr-image или qrcode. Они предоставляют удобный интерфейс для создания и чтения QR-кодов в различных форматах.e
    • Для обеспечения безопасности: Для обеспечения безопасности можно использовать различные библиотеки, в зависимости от конкретных требований и угроз. Например, для защиты от XSS-атак рекомендуется использовать библиотеку Helmet, для защиты от DoS-атак можно использовать библиотеку express-rate-limit, а для проверки входных данных - express-validator.
  • API:
    • Стандарт для работы с API: RESTful API
    • Для авторизации и аутентификации пользователей можно использовать токены JWT.
    • Для просмотра истории операций и управления статусами операций можно использовать базу данных и соответствующие запросы.
  • Архитектура:
    • Фронтенд и бэкенд разделены на две части, связанные между собой через API.
    • Для обеспечения безопасности веб-приложения рекомендуется использовать SSL/TLS протоколы для защиты данных, передаваемых между клиентом и сервером.
  • Описание логики работы сервиса:
    • Организация регистрируется в качестве участника СБП через форму регистрации, которая отправляет запрос на бэкенд, где происходит валидация данных и сохранение в базе данных.
    • Пользователь может авторизоваться в системе с помощью логина и пароля или токена JWT, после чего ему будет доступен функционал генерации QR-кода для оплаты.
    • При генерации QR-кода, в него включается необходимая информация об операции, включая сумму, идентификатор организации и другую информацию.
    • Сгенерированный QR-код предоставляется пользователю на экране или отправляется на email или мобильный телефон.
    • При оплате, система проверяет корректность QR-кода и проводит транзакцию. После того, как транзакция проведена, статус операции меняется на "выполнено". Если транзакция не может быть выполнена по каким-либо причинам, статус операции меняется на "отклонено".
